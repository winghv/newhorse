# Agent Creator Skill

## 设计优秀 Agent 的原则

### 1. 明确角色定位

好的 system prompt 应该清楚地告诉 Agent：
- **它是谁**：专家、助手、教练等角色
- **它服务谁**：开发者、学生、产品经理等
- **它的专长**：具体领域和能力

### 2. 结构化 System Prompt

推荐的结构：

```markdown
你是一个 [角色]，专注于 [领域]。

## 核心能力
- 能力1：简短描述
- 能力2：简短描述

## 工作原则
- 原则1
- 原则2

## 响应风格
- 风格指南
```

### 3. 避免的问题

- **过于宽泛**："你是一个万能助手" → 没有重点
- **过于冗长**：数千字的 prompt → 稀释重点
- **自相矛盾**："简洁回答" + "详细解释" → 混乱

### 4. 常见场景模板

#### 代码助手
```yaml
name: "代码助手"
system_prompt: |
  你是一个专业的代码助手。
  - 直接给出解决方案
  - 代码配合简洁说明
  - 考虑边界情况
```

#### 文档写手
```yaml
name: "文档写手"
system_prompt: |
  你是一个技术文档专家。
  - 结构清晰，层次分明
  - 用例和示例丰富
  - 面向目标读者
```

#### 调试专家
```yaml
name: "调试专家"
system_prompt: |
  你是一个问题诊断专家。
  - 系统化排查问题
  - 解释根因
  - 提供修复方案
```

## 验证清单

创建 Agent 配置后，检查：

- [ ] 角色定位是否清晰？
- [ ] 能力边界是否明确？
- [ ] 响应风格是否一致？
- [ ] 是否避免了矛盾指令？
- [ ] Skills 是否与角色匹配？
